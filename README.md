# Haru Hanjul (하루 한줄) - 일일 감성 글귀 서비스

## 기간
2025.03 ~ 2025.03

## 프로젝트 개요
🌟 하루에 한 줄, 마음을 채우는 글귀 🌟

매일 감성적인 글귀를 제공하는 커뮤니티 기반 서비스
- 매일 랜덤 글귀 노출
- 유저 참여형 콘텐츠 (글귀 업로드 → 운영자 승인)
- 게이미피케이션 요소 (로그인 포인트, 스트리크, 복권)

## 기술 스택
- Backend: Python, Django, Django Rest Framework
- Authentication: JWT, OAuth2
- Frontend: Flutter (Web 배포)
- Hosting: AWS / Cloudflare

## 구현한 기능
- ✅ 글귀 랜덤 노출 시스템

## 기술적 성과
- Django + DRF로 RESTful API 설계
- JWT/OAuth2 인증 구현
- Flutter Web 배포 경험

## 왜 멈췄는가
- **성능 이슈**: Flutter Web이 너무 느림
- React 전환 or 앱 배포 필요 → 추가 공수 큼
- 유저 관련 작업 중지: 콘텐츠 플랫폼은 초기 유저 없으면 의미 없음
- 혼자 운영하기엔 콘텐츠 관리 부담

## 배운 것
- Flutter Web의 한계 체감 (초기 로딩 느림, SEO 불리)
- 커뮤니티 플랫폼은 콘텐츠보다 **사람**이 먼저
- Django + DRF 실전 경험 축적

## 회고
- 기술적으로는 많이 배웠지만, 비즈니스적으로는 실패한 프로젝트.
- "좋은 아이디어 ≠ 성공하는 서비스"를 몸소 체험.
- Flutter Web 성능 이슈를 미리 검증하지 못한 게 아쉬움.

## 향후 가능성
- React 전환하면 성능 문제 해결 가능
- 앱으로 배포하면 UX 개선 가능
- 하지만 **지금은 우선순위 아님**


# Haru Hanjul (하루 한줄)

🌟 하루에 한 줄, 마음을 채우는 글귀 🌟
🌟 One line a day, filling your heart with words 🌟  

**하루 한줄**은 매일 한 줄씩 감성적인 글귀를 제공하는 서비스입니다.  
유저는 직접 글귀를 업로드할 수도 있으며, 운영자의 검토 후 승인된 글귀가 랜덤으로 배포됩니다.  
또한, 로그인 포인트, 스트리크 보상, 포인트 복권 등 다양한 보상 시스템을 통해 유저 참여를 독려합니다.

## 주요 기능
- 매일 한 줄의 글귀 랜덤 노출 📜
- 유저 글귀 업로드 및 승인 시스템 ✍️
- 로그인 포인트 및 스트리크 보상 🎁
- 포인트를 활용한 복권 이벤트 🎟️

## 기술 스택
- **Frontend**: 미정
- **Backend**: Python, Django, Django Rest Framework
- **Authentication**: JWT, OAuth2
- **Hosting**: AWS / Cloudflare
